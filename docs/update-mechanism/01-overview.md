# ClawX 应用更新机制

## 概述

ClawX 是一个基于 Electron 的跨平台桌面应用，使用 `electron-updater` 实现自动更新功能。更新系统采用双源策略：主要使用阿里云 OSS 提供更新，GitHub Releases 作为备用源。

## 架构概览

## 核心组件

| 组件 | 位置 | 职责 |
|------|------|------|
| `AppUpdater` | `electron/main/updater.ts` | 核心更新逻辑 |
| `useUpdateStore` | `src/stores/update.ts` | 前端状态管理 |
| `UpdateSettings` | `src/components/settings/UpdateSettings.tsx` | UI 组件 |
| 配置 | `electron-builder.yml` | 发布配置 |

## 更新流程

1. **应用启动**: 初始化 `AppUpdater`，注册 IPC 处理器
2. **自动检查**: 如果启用自动检查，10秒后开始检查更新
3. **检查更新**: 请求 OSS/GitHub 获取最新版本信息
4. **下载更新**: 用户确认或自动下载更新包
5. **安装更新**: 下载完成后自动或手动安装

## 通道 (Channels)

- **stable**: 稳定版
- **beta**: 测试版
- **dev**: 开发版

版本号示例：
- `0.1.23` → stable
- `0.1.8-alpha.0` → alpha
- `1.0.0-beta.1` → beta
